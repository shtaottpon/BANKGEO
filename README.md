# УКРСИББАНК · Автоматичний розподіл нових клієнтів між відділеннями

Інструмент для співробітників банку, які щоранку вивантажують з vkursi.pro
список новостворених ФОП/ЮО у Вінницькій області та вручну розподіляють їх
між 6 відділеннями (Вінниця, Калинівка, Козятин, Бершадь, Гайсин, Немирів).

**Було:** ~60 хв ручної роботи + людські помилки.
**Стало:** 1 кнопка → 1–2 хв → готовий Excel і карта.

## Як працює (конвеєр)

```
Excel з vkursi.pro
       │
       ▼
[1] Читання + очистка адрес  ←  regex-нормалізація скорочень
       │
       ▼
[2] Геокодування (Nominatim/OSM)
       │   ↳ fallback: повна адреса → лише населений пункт → ручна перевірка
       ▼
[3] Формула гаверсинуса — відстань до 6 відділень
       │
       ▼
[4] Розподіл на найближче (з порогом для дистанційного обслуговування)
       │
       ▼
Excel з розподілом + інтерактивна карта
```

## Безпека даних

У зовнішній сервіс (Nominatim) надсилається **тільки адреса**.
Назва клієнта, ЄДРПОУ, телефон та інші персональні дані залишаються
на комп'ютері банку. Nominatim — open-source, його можна підняти
**локально на сервері банку**, і тоді жодного зовнішнього виклику не буде взагалі.

## Запуск (3 кроки)

```bash
# 1. Встановити залежності (одноразово)
pip install -r requirements.txt

# 2. Запустити веб-інтерфейс
streamlit run app.py

# 3. Відкрити http://localhost:8501 → завантажити Excel → натиснути «Розподілити»
```

### Або з командного рядка (без UI):

```bash
python geocoder.py test_clients.xlsx -o result.xlsx --threshold 15
```

## Структура файлів

| Файл | Призначення |
|---|---|
| `geocoder.py` | Ядро: читання, геокодування, гаверсинус, розподіл |
| `app.py` | Streamlit-інтерфейс з картою та статистикою |
| `branches.csv` | Довідник 6 відділень УКРСИББАНКу з координатами |
| `test_clients.xlsx` | Тестовий файл на 38 клієнтів для демо |
| `requirements.txt` | Залежності Python |

## Очікувані метрики на демо

- **Геокодовано:** ≥ 90% адрес з типового файлу vkursi.pro
- **На ручну перевірку:** ≤ 10% (порожні/криві адреси, віддалені села)
- **Швидкість:** ~1 сек/клієнт (ліміт публічного Nominatim)
- **Можна прискорити в 10–100× розгорнувши Nominatim локально**
